In-flight cap enforcement

Target: make a running agent spend breach stop while it is still running, then surface the breach through the durable notification outbox.

Scope: runtime metering, process termination, ledger stats, status projection, proof fixture, docs/backlog notes. No provider-key changes and no plane/ excision in this slice.

Acceptance

A deliberately over-cap run emits live usage, BB records partial cost, kills the process before normal completion, marks the run failed, and delivers a run_in_flight_cap_killed notification through the outbox.

Design
  • Add an execution monitor at the substrate boundary.
  • Parse partial harness usage from JSONL stdout while the process runs.
  • Update attempt stats before completion so bb status --json can see spend.
Enforcement
  • Use task max_cost_per_run_usd as the in-flight threshold.
  • Use agent policy.side_effect_policy for action; default to kill.
  • Kill local process groups and Sprite marker process groups.
Proof
  • Red test in tests/budgets.rs.
  • Local dev-plane live drill with notify stub.
  • Repo gate: ./scripts/verify.sh.

Stop Conditions

Stop before merge if the monitor cannot kill a sleeping process before timeout, if notification outbox delivery is not visible, if running status cannot show partial cost, or if this requires workload-specific logic in the Rust spine.
