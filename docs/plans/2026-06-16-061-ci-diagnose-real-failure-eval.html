Backlog 061 / SDLC lifecycle reflex pack

Turn CI diagnose from a no-failure probe into a real failed-run, model-evaluated reflex receipt.

The current plane has ci-diagnose, Kimi and GLM variants, and model-eval. This slice proves the loop on a real failed GitHub Actions payload, records which diagnoser output is most useful, and tightens the operator contract discovered by dogfood.

Chosen Design

Use the existing data-owned task cards and agent configs. Add only the missing reference record, docs/tests needed to make the record durable, and any prompt/contract fix forced by the run evidence.

No Rust workflow engine, no auto-builder launch, no GitHub comments, no merge authority.

Acceptance

  • A real failed CI/check-suite payload is selected from live GitHub state.
  • ci-diagnose, ci-diagnose-kimi, and ci-diagnose-glm run on the same payload.
  • model-eval compares the three reports and writes an evaluator receipt.
  • A new docs/model-evals/ci-diagnose/ reference record names run ids, models, costs, diagnosis quality, winner, and residual risk.
  • 061 is updated to distinguish completed CI-diagnose evidence from remaining lifecycle work.

Verification

  • cargo test --test lifecycle_reflex -- --nocapture
  • cargo test --test model_eval -- --nocapture
  • ./scripts/verify.sh
  • Live bb candidate/evaluator run ids and costs.
  • Fresh adversarial review against diff plus this oracle.

Stop If

  • GitHub auth or OpenRouter/Sprite auth blocks real runs.
  • The three candidate reports are malformed; score them as failed evidence, do not hide them.
  • The selected CI failure is not reproducible from GitHub logs.
  • The change wants workload-specific Rust routing.
