# Context Packet: Task Inventory CLI

## Goal

Add a first-class `bb task list` command so operators and agents can inspect
loaded tasks and parked state from the CLI.

## Non-Goals

- Do not change task loading, dispatch, budget, parking, or serving
  semantics.
- Do not add a new long-lived daemon or depend on `/api/tasks`.
- Do not replace `bb check`; validation remains its job.

## Constraints

- The Rust spine stays small and generic.
- The CLI output must be useful to humans and stable enough for agents via
  `--json`.
- Parked state must come from the ledger, not from config.
- Repo-owned task source metadata must remain visible when present.

## Alternatives

- **Keep using `bb check`: rejected.** `check` validates config and prints a
  mixed agent/task summary; using it as inventory couples operator scripts to
  validation prose and omits parked state.
- **Use `/api/tasks`: rejected as the only path.** It has the right data but
  requires `bb serve`, auth choices, and a network hop for a local operator
  question.
- **Add `bb task list`: selected.** It keeps the operator surface local,
  mirrors the read API data, and fits the existing `task park/unpark`
  namespace.

## Design

Add `TaskCommand::List { json }`. It loads the already-validated `Plane`,
queries `Ledger::parked_reason` for each task, and emits rows containing:

- `task`
- `agent`
- `harness`
- `model`
- `substrate`
- `triggers`
- `verdict`
- `source`
- `parked`

Text mode prints one compact task object per line. JSON mode prints a
pretty array of the same objects.

## Oracle

```bash
cargo test task_list
./scripts/verify.sh
./target/debug/bb --config plane task list
./target/debug/bb --config plane task list --json | jq '.[] | select(.task=="review")'
```

Acceptance:

- Text output includes every loaded task and `parked` field.
- JSON output includes `parked` as `null` or the parked reason.
- A test creates a parked task and proves both the task row and parked reason
  are present through the actual `bb task list --json` command.

## Premise Source

Waiver: the premise came from the live operator session rather than a durable
incoming ticket. Evidence command:
`./target/debug/bb --config plane task list` failed with
`unrecognized subcommand 'list'` on 2026-06-13.

## Risks + Rollout

Risk is low: this is a read-only CLI surface over data the plane already
loads and the server already exposes. Rollback is deleting the command and
test. The main failure mode is unstable text output being mistaken for a
machine contract, so JSON is the supported agent surface.
