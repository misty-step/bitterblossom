Strategic themes

The backlog condenses into five compounding tracks. The first three are the actual MVP spine.

Remote control-plane reliability Run the conductor away from the operator shell and make lease ownership self-healing.
Isolated execution + truthful observability Per-run worktrees and run-centric status surfaces keep state explainable and recoverable.
Intake and routing quality Only machine-runnable issues enter the factory; ranking uses semantic reasoning, not brittle heuristics.
Experimental breadth Parallel variants stay explicit and measurable rather than becoming default chaos.
Direct dispatch hardening Legacy `bb dispatch` improvements stay in the backlog but no longer define system direction.

Execution sequence

If Bitterblossom is going to clear the backlog itself, this is the right order.

1. Stand up the remote conductor and make leases recoverable.
2. Isolate every run and expose truthful run events.
3. Prove the trace bullet end-to-end on a target repo.
4. Tighten intake and semantic routing so only runnable issues flow in.
5. Expand with Sentry intake and parallel variants after the core loop is trustworthy.
