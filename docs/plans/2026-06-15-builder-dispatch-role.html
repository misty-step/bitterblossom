Builder Dispatch Role

Deliver the first manual authoring lane for Bitterblossom without turning the Rust spine into a semantic workflow engine.

Goal

Add a versioned, manual-only Rust builder launch contract and task card so an operator can dispatch focused code-authoring work through bb run build, then verify the resulting branch through the existing submission/gate loop.

In Scope

  • Agent metadata for role and curated skill/profile references.
  • A production plane bb-builder-rust subscription agent.
  • A manual-only build task card with branch/report/no-merge boundaries.
  • Docs and tests proving the metadata parses and is visible from task JSON.

Out Of Scope

  • No multi-step orchestrator inside dispatch.
  • No runtime skill projection or Daedalus-generated agent deployment yet.
  • No reflex/webhook builder task; subscription agents stay manual dispatch only.
  • No automatic merge authority for builder runs.

Acceptance Oracle

CheckEvidence
Config contractrole and skills load, validate, and appear in task JSON.
Builder taskbb --config plane check --json lists build bound to bb-builder-rust@v1.
Repo gate./scripts/verify.sh passes without weakening lint/test/LOC gates.
DogfoodA local bb submit/bb gate --json storm is attempted and UX notes are recorded.

Primary Risk

The easy mistake is to build a hidden meta-agent framework. The intended change is narrower: make role/skill metadata first-class enough for operators and future Daedalus contracts while leaving execution as the existing card plus harness command.
