# Prove the Estate execution boundary

This is a deterministic command-harness integration proof. The command is
fixed in the plane-owned `hephaestus` binding; no model interprets the plan and
no event selects an identity or command.

Run Estate's external signed-fixture proof from the declared immutable commit
with its declared `Cargo.lock` Git blob. Contact no provider. Return
`ESTATE_EXECUTION_EVIDENCE.json`; Bitterblossom owns only isolation, lifecycle,
budgets, and evidence retention.

## Boundaries

A refused credential is a boundary, not a puzzle: on HTTP 401/403 (or any
authorization refusal) from a credential this run declares, STOP-and-report —
write `REPORT.json` naming the refused operation and the refused credential by
name (never its value), then stop without completing the goal. Never locate or
use a stronger credential (env, keychain, 1Password, config, another agent).
