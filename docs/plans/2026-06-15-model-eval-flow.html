Change Shape

  • Use symlinked task cards so variant tasks share one checked-in card without copy-paste or Rust spine changes.
  • Add Kimi and GLM CI-diagnose task variants using the existing CI diagnose card.
  • Add a manual model-eval task that evaluates candidate run reports and writes REPORT.json.
  • Add durable reference docs under docs/model-evals/ and operator recipe updates.
  • Add tests that enforce at least three diverse configs plus evaluator for the first flow.

Verification Loop

  1. Red tests for missing model-eval cohort, model diversity, and durable reference docs.
  2. Green config/tests with cargo test --test model_eval, cargo test --test policy, and bb --config plane check.
  3. Dogfood: run all three CI-diagnose candidate tasks against the same known GitHub Actions run, then pass report snippets into bb run model-eval.
  4. Record evaluator output in docs/model-evals/.
  5. Full ./scripts/verify.sh, then fresh-context review via bb storm.

Stop Conditions

  • Stop and redesign if shared-card reuse requires workload-specific Rust logic.
  • Stop if GLM 5.1 cannot run through Pi/OpenRouter; replace with another current non-DeepSeek/non-Kimi comparator and document why.
  • Do not unpark the existing security verdict task merely to make a gate fully clear.
