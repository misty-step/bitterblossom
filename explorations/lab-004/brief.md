# LAB-004 fence and lane contract

Artifact: Bitterblossom's authenticated workflow-control-plane application.

Intent: rerun the complete workflow-first design spine while treating Misty Step
Aesthetic v0.25.0 as binding product law, not a mood board. The result must let
an operator understand configured workflows, topology, agents, live instances,
authority, spend, evidence, and goal-first creation without reading logs.

## Fixed

- Product truth and nouns in `VISION.md` and `docs/workflow-control-plane.md`.
- Misty Step Aesthetic v0.25.0 is the only visual vocabulary. Load its pinned
  stylesheet and use its tokens, recipes, and `ae-*` component costumes.
- Aesthetic invariants: one size per surface plus 13px chrome, nine ink/weight
  registers, hairlines, radius 0, no decorative shadows or gradients, status on
  glyphs, motion only as resolving feedback, buttons are not links, cursor law.
- Light and dark are equal and default to system; the shared mode control must
  work in every option. WCAG AA, keyboard reach, and reduced motion are required.
- Primary destinations: Workflows, Agents, Runs, Spend, Create workflow.
- The landing answers what is configured. A workflow detail separates stable
  configuration from selected-run evidence and distinguishes workflow state,
  trigger health, run lifecycle, domain result, verification, and cost truth.
- The same truthful `CORPUS` renders in every candidate. No fake production
  claims, remote imagery, marketing copy, or second design system.
- Full-viewport responsive pages at desktop, tablet, and 390px phone.

## Varying

- Information architecture inside the required destinations.
- Roster/detail relationship, topology grammar, progressive disclosure, density,
  ordering, hierarchy, composition, navigation placement, and responsive model.
- Aesthetic consumer dials: accent frequency, mono ratio, measure, chrome
  density, and how much structure is visible at once.

## Dials

- VARIANCE: 9/10 structural, 0/10 off-system styling.
- MOTION: 2/10, direct and resolving only.
- DENSITY: 7/10, with at least one high-density proposition per lane.

## Required gallery inside every option

Every candidate must expose, through visible composition or working view controls:

1. configured workflow roster;
2. PR Review topology and a selected live-run overlay;
3. agent roster with In use and Available separated;
4. live/history evidence;
5. spend controls and reported/estimated/unavailable truth;
6. goal-first creation with enhanced-goal review and fixture test.

Required states: active, draft, listening trigger, executing, succeeded execution
with blocked domain result, superseded, verification achieved, unavailable cost,
and no active run. Never collapse these into one traffic-light status.

## Lane output

Write exactly one file: `explorations/lab-004/lanes/<alias>.js`. Export `SPECS`,
an object with three namespaced IDs. Each value has `label`, `move`,
`philosophy`, and `render(corpus)`, returning a complete HTML string. Use only
Aesthetic tokens and classes for visual vocabulary; option-scoped CSS may own
layout but may not introduce raw colors, radii, shadows, type scales, or motion
curves. Every option includes `[data-theme-toggle]` and uses the shared corpus.

Return three structurally distinct propositions, not skins. One must invert a
load-bearing layout assumption. End with `ID — structural move` per option.

## Registry adjudication

Six blind philosophies produced 18 raw systems. The round-one catalog retains
12 candidates plus the shipped baseline. Six raw systems were excluded before
operator review for deterministic contract failures and structural duplication:

- `TASTE-1` duplicated the fixed-rail control desk expressed with more complete
  shared-corpus coverage by `ANTH-1` and `BRUT-1`.
- `TASTE-2` duplicated topology-first navigation in `BRUT-2` and `APPLE-2`.
- `TASTE-3` duplicated the evidence/run-first inversion in `MIN-2` and `APPLE-3`.
- `IMPEC-1` duplicated the roster plus selected-run rail in `ANTH-1` and
  `APPLE-1`.
- `IMPEC-2` duplicated the configured/in-motion/evidenced ledger split in
  `BRUT-1` and `BRUT-2`.
- `IMPEC-3` duplicated the evidence-first inversion in `MIN-2` and `APPLE-3`.

The Taste modules substituted fallback agent data instead of rendering the
shared Cerberus/Verifier corpus; the Impeccable modules omitted the required
Canary Resolution state. No candidate was excluded for palette or taste.
