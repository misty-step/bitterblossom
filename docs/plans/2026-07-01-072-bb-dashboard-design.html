Backlog 072 / observability core + dashboard

Bitterblossom gets a read-mostly operator dashboard over the existing plane.

The dashboard is drill-down visibility, not a control plane. Every datum must come from the CLI/API read surface: status, tasks, runs, DLQ, leases, ingress activity, and run export. The HTML stays static and consumes JSON.

Chosen Design

■ Finish the existing operator.html shell served by bb serve.

■ Add thin read mirrors where the backlog names gaps.

■ Keep workload judgment out of Rust and out of the UI.
