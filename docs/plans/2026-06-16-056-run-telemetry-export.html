Backlog 056

Run Telemetry Export v1

Turn bb runs export from a raw ledger dump into a stable, versioned JSONL contract that Daedalus can ingest and OTel GenAI adapters can map later, without adding a sidecar or workload logic to the Rust spine.

User
Operator and lab tooling
Surface
bb runs export
Default
Versioned JSONL envelope
Done
Docs, fixture, schema test, live dogfood

Chosen Shape

  • Keep bb runs export as the integration seam.
  • Emit one bb.run_telemetry.v1 JSON object per ledger run.
  • Normalize run, attempts, retry/DLQ, artifacts, Daedalus, and OTel mapping hints.

Current Gap

  • Current output is unversioned raw {"run": ..., "attempts": ...}.
  • No compatibility rules, fixture, Daedalus handoff, or OTel field map.
  • Operators cannot tell which fields are contract versus incidental ledger internals.

Constraints

  • src/ is under a strict 5000 LOC gate.
  • No OTel/Langfuse sidecar in the default spine.
  • Dogfood must exercise real bb commands, not just tests.

Contract Fields

AreaContentsReason
Envelopeschema, schema_version, exported_atConsumers can pin a parser and reject unknown major versions.
Runid, task, trigger, state, timestamps, duration, cost, total tokens, trace id, errorOne row carries the operator-level outcome and budget signal.
Attemptsattempt number, phase, outcome, agent name/version, harness, model, cost, tokens, artifact pointerDaedalus can compare agent configs and map failures to launch contracts.
Retry/DLQattempt count, retryable status, dead-letter status and replay pointer when presentFailure accounting survives outside the plane.
Artifactstyped path pointers to run and attempt outputsDownstream evaluators can fetch evidence without guessing ledger internals.
Daedalustask key, agent config keys, score placeholders, source run pointerDefines the handoff without importing Daedalus code.
OTelGenAI span and metric attribute mapping hintsFuture adapters can emit standard telemetry without changing the export contract.

Execution

  • Document docs/run-telemetry-export-v1.md first.
  • Add a stable fixture under tests/fixtures/.
  • Add CLI/schema tests that parse real bb runs export output.
  • Implement the versioned envelope with minimal spine LOC.
  • Move backlog item to done only after verification and review.

Dogfood Loop

  • Run the old export once to capture baseline friction.
  • Run the new export against plane/ and parse sample rows.
  • Use bb submission/review gates if available on this branch.
  • Record what felt good, bad, or missing in the doc or closeout.

Review Focus

  • Is the schema deep enough for Daedalus without coupling repos?
  • Are OTel fields mapped but not overbuilt into a runtime sidecar?
  • Does compatibility language protect consumers?
  • Does the code stay inside the spine budget?

Stop Conditions

  • ./scripts/verify.sh fails after two distinct remediation approaches.
  • The new contract needs Daedalus code changes to be meaningfully specified.
  • Live dogfood reveals the existing plane ledger cannot provide a required oracle field.
