Backlog 063 / model catalog watch

Make model freshness a deterministic guard plus a report-producing reflex.

The target outcome is not "use whatever launched today." The outcome is that configured OpenRouter model ids stay valid, new runnable models become visible quickly, and default promotions still require Bitterblossom smoke plus model-eval evidence.

Chosen Design

Build a hybrid: an offline catalog fixture check in the repo gate, plus a scheduled model-catalog-watch task that fetches the live OpenRouter catalog and writes recommendation artifacts.

Do not add provider polling to the Rust spine in this slice. The current src budget has only 19 LOC of headroom.

Standards

Deterministic local gate, no silent config mutation, event-plane workload for live discovery.

Acceptance

Configured models validate against a fixture; the live watcher produces REPORT.json recommendations.

Stop If

The implementation wants live network in default verify, Rust provider polling, or automatic model promotion.

Current State

  • plane/agents/*.toml hold harness, provider, and model ids.
  • docs/model-evals/README.md owns promotion doctrine.
  • scripts/verify.sh is the one local gate and enforces src LOC <= 5000.
  • OpenRouter exposes model metadata via GET /api/v1/models.

Proof Surface

  • cargo test --test model_catalog -- --nocapture
  • scripts/check-model-catalog.sh --catalog tests/fixtures/openrouter-models-current.json --json
  • scripts/check-model-catalog.sh --live --json
  • bb --config plane run model-catalog-watch --payload '{"dry_run":true}' --json
  • ./scripts/verify.sh

Execution Plan

1. Guard

Add a shell catalog check with fixture and live modes. It reads configured OpenRouter models and fails on missing ids or required metadata gaps.

Evidence: fixture test plus JSON output that names configured and missing ids.

2. Reflex

Add model-catalog-watch task/agent/card with manual and cron triggers. The card writes recommendations and may file one backlog PR when explicitly not dry-run.

Evidence: bb check sees the task and a dry-run produces REPORT.json.

3. Doctrine

Document that release pages and RSS are discovery signals, while API catalog plus smoke plus model-eval is the promotion path.

Evidence: model-eval docs name the policy and fixture boundary.

4. Close

Run full gate, live probe, and fresh artifact-only review focused on provider flake, silent mutation, and spine creep.

Evidence: green commands, watcher run id, clean git state.

Alternatives

OptionHelpsFailsVerdict
Manual sweep Cheap, flexible. Depends on memory and misses same-day launches. Fallback only.
Pure LLM watcher Fits Bitterblossom reflex style. Too soft for configured-id correctness. Use only as recommendation layer.
Live CI check Reality on every run. Provider outages block unrelated work. Reject for default gate.
Rust bb model-catalog First-class CLI. Blows or strains the 5k spine budget unless paired with refactor. Defer.
Offline guard plus live reflex Deterministic safety and durable discovery. Requires fixture refresh discipline. Choose.
